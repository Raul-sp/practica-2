# Explicación de comandos basicos de Git
**git init**: Inicializa un nuevo repositorio de Git en el directorio actual, creando una carpeta oculta llamada .git.

**git add**: Añade cambios de archivos al área de preparación (staging area), preparándolos para el próximo commit. Se usa especificando el archivo (git add archivo.txt) o todos los cambios (git add .).

**git commit -m "mensaje"**: Crea un commit de los cambios en el área de preparación, guardando un estado del proyecto con el mensaje que describes entre comillas.

**git push**: Envía los commits locales al repositorio remoto, sincronizando los cambios realizados en tu máquina con el repositorio compartido.

**git pull**: Trae y fusiona los cambios del repositorio remoto al repositorio local, actualizando los archivos en tu máquina.

**git remote -v**: Muestra la lista de repositorios remotos configurados para el proyecto, con sus respectivas URLs y nombres (por ejemplo, origin).
